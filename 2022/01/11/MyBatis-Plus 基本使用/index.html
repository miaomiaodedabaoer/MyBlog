    • MyBatis 特点

    SQL 语句自由控制

    代码分离

    提供 XML 标签，编写动态 SQL

    简单的 CURD 也要写 SQL

    XML 中大量SQL维护

    自身功能有限（物理分页也没有，），

    JPA （标准 hibernate, Spring Data JPA）

    移植性好 ( JPQL )语句

    提供很多 CRUD 方法，不用写 SQL

    对象化程度高

    MyBatis-Plus

    只增强，不改变

    无侵入，损耗小（启动时就注入），强大的 CRUD

    支持 Lambda 形式调用，多种数据库

    主键自动生成，支持 ActiveRecor

    全局通用操作，支持关键词自动转译

    代码生成器，内置分页，性能分析插件

    全局拦截插件，内置 Sql 注入剥离器

    SSM 传统编程模式

    1. 接口中写抽象方法
    2. XML 或者 注解中写 SQL
    3. Service 中调用接口
    4. Controller 中调用

    常用注解

    不允许 null 的字段就不写在MP内置的 sql 语句中

    默认数据库下划线，对应驼峰命名；

    默认主键叫 id；

    @TableName(“XXX”) 指定表名

    @TableId 指定主键

    @TableField(“name”) 指定属性

    排除非表字段的三种方式（临时数据等）

    1. transient 标识，不参与序列化过程
    2. static 标志为静态变量
    3. @TableField(exist=false) 表示不是数据库中字段，默认是 true
